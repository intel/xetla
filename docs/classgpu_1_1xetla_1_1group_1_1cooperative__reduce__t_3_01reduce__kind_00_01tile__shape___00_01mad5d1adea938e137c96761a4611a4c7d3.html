Cooperative workgroup reduction.

1) each workgroup stores tile data to local memory -> 2) cross workgroup (but still within a group) sync -> 3) each workgroup loads slice of tile data, do the reduction.

Parameters
gIs the workgroup of the current tile.
mat_sliceIs the output of the reduction. Each workgroup only keeps part of the tile data.
matAccIs the input of the reduction.
slm_baseIs the slm base address.
nbarrier_baseIs the named barrier base.
